### Sync-invocation note (root cause, systematic-debugging)

The planned command `sync --out <path>` (space-separated) errored with
`The "paths[0]" argument must be of type string. Received type boolean (true)`.
Root cause (layer = bin/concertino arg parsing): `parseArgs` (bin/concertino:104-112)
only recognizes `--flag=value`; a space-separated `--out <path>` sets `args.out = true`
and the path lands in positional `_`. Probe: monkeypatched `path.resolve` to log
non-string args — confirmed `path.resolve(true)` at `cmdSync` line 1202. Fix: invoke
with the `--out=<path>` form. This is a pre-existing CLI ergonomic, not a sync/render
failure; sync rendered correctly once given the `=`-form, so no files were hand-forged.
