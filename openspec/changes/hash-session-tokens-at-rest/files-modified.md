# Files Modified — hash-session-tokens-at-rest (HEL-288)

- `backend/src/main/scala/com/helio/infrastructure/TokenHashing.scala` — new shared `sha256Hex` helper (extracted from `ApiTokenService.sha256Hex`), used by both credential kinds (api_tokens and user_sessions).
- `backend/src/main/scala/com/helio/services/ApiTokenService.scala` — `sha256Hex` now delegates to `TokenHashing.sha256Hex`; no behavior change, `AuthDirectives.resolveApiToken`'s call site untouched.
- `backend/src/main/resources/db/migration/V45__hash_session_tokens.sql` — new migration: `DELETE FROM user_sessions;` then renames column `token` → `token_hash` and constraint `user_sessions_token_unique` → `user_sessions_token_hash_unique`.
- `backend/src/main/scala/com/helio/infrastructure/UserRepository.scala` — `SessionRow.token` → `tokenHash`; `SessionTable` maps `tokenHash` to the `token_hash` column; `createSession` hashes the raw token before insert but returns the original raw-token `session` unchanged; `findSession`/`deleteSession` hash the incoming raw token before querying, and `findSession`'s returned `UserSession` carries the raw token param back (not the row's hash).
- `backend/src/main/scala/com/helio/infrastructure/UserSessionRepository.scala` — `SlickUserSessionRepository.findValidSession` hashes the incoming raw token before filtering on `tokenHash` (the hot per-request auth path).
- `backend/src/test/scala/com/helio/api/ApiRoutesSpec.scala` — added a `"user_sessions token hashing"` block: (1) asserts the persisted `token_hash` is the SHA-256 hex digest of the raw token, never the raw value; (2) a repository-level round-trip test for `createSession`/`findSession`/`deleteSession`/`findValidSession` confirming a raw token presented at lookup time correctly matches its hashed row. Existing register/login/logout/`GET /api/auth/me` tests (real-session-repo path) were left unmodified and pass as-is, since they only round-trip through `AuthService`/`AuthResponse.token` and never assert on the raw column value.

No frontend changes — the client-facing `Authorization: Bearer <token>` contract and `AuthResponse.token` shape are unchanged; only the at-rest DB representation changes. Confirmed no `frontend/` code references `user_sessions` or the session token's storage shape.
