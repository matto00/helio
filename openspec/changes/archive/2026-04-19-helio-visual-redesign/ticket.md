# HEL-62: Helio Visual Redesign

## Context

Helio has a functional UI but lacks a distinctive visual identity. The goal is to make the app immediately recognizable and visually compelling — something that stands out from generic dashboard tools and communicates quality to users on first impression.

## Approach

This is a design-exploration ticket. Use the Claude AI design skill to prototype and evaluate multiple style directions (color palettes, typography choices, component aesthetic, spacing system, overall visual language) before committing to an implementation. Iterate until a direction clearly stands out as both distinctive and production-appropriate.

## Scope

* Color palette (primary, accent, surface, text, border tokens)
* Typography (font choice, scale, weight system)
* Component styling (buttons, inputs, cards, modals, nav)
* Spacing and layout system
* Overall aesthetic direction (e.g. minimal, bold, dark-first, glass, etc.)
* Side nav experience may be included or follow as a dependent ticket

## Acceptance Criteria

- [ ] At least 3 distinct visual directions are prototyped and compared
- [ ] A single winning direction is selected and documented (palette, fonts, key component styles)
- [ ] Core app shell (sidebar, topbar, dashboard grid, panel cards) is restyled to match the chosen direction
- [ ] Both light and dark themes are updated consistently
- [ ] The result looks clearly different from the current generic styling
