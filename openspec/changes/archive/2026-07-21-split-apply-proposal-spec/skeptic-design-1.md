### Verdict: REFUTE

### Change Requests
1. Align the file-size acceptance criterion with CONTRIBUTING.md's actually-enforced soft budget. `CONTRIBUTING.md:24` states the soft budget is "~250 lines per source file"; ~400 lines is only the "propose a split" trigger point, not the target size after splitting. `scripts/check-scala-quality.mjs` (run via `npm run check:scala-quality`) warns at 250 lines and scans `backend/src/test/scala`. Update `specs/backend-file-size-compliance/spec.md`'s scenario ("comfortably under ~400 lines"), `design.md`'s Goals bullet, and `tasks.md` task 4.1 to target "comfortably under the ~250-line soft budget," and have 4.1 actually run `npm run check:scala-quality` to confirm zero new soft warnings for the split/base-trait files, rather than an eyeballed 400-line check.
2. Correct `design.md`'s Decision section ("Introduce an abstract base trait ... that holds the imports, `beforeAll`/`afterAll`, seeded-id vars, and the private helpers as `protected` members") and the matching `tasks.md` 2.1. Import statements are file-scoped in Scala and are never inherited by subclasses defined in other files, so each new sibling spec file will still need its own top-of-file imports for symbols used directly in moved test bodies (e.g. `StatusCodes`, `JsObject`/`JsBoolean`, `UUID`). Reword so the base trait's shared surface is scoped to `beforeAll`/`afterAll`/seeded-id vars/request helpers only, and note per-file imports are expected, ordinary boilerplate — not something the trait eliminates.

### Non-blocking notes
- The "V41 companion-binding + unknown/cross-user rejections" sibling grouping (`tasks.md` 3.2) conceptually overlaps with tests interleaved inside the HEL-316 section (lines 508, 565, 578, 635), which `design.md`'s own bullet list places under the HEL-316 config-parity group instead. This is genuinely ambiguous but harmless: `design.md` explicitly grants the executor discretion to refine grouping while keeping cohesion + under-budget, and the mechanically-checked ACs (line count, `sbt test` total) don't depend on which sibling file a given case lands in.
