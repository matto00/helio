## Root cause (systematic-debugging)

- **Root cause:** `PanelMutationOps.batchUpdate`'s config-patch branch (`PanelMutationRepository.scala:105-108`, pre-fix) wrote a hand-written Slick column tuple that omitted the `aggregation` column added in V43/HEL-292, so a patched aggregation spec was computed correctly in memory but silently dropped on write in the infrastructure layer.
- **Probe:** wrote the two regression tests, then `git stash push` on the two source files (leaving only the new tests), and ran `sbt "testOnly com.helio.api.ApiRoutesSpec -- -z \"HEL-296\""`.
- **Probe output:** both tests failed with `java.util.NoSuchElementException: key not found: aggregation` (the batch-path response's `config` JSON object had no `aggregation` key), confirming the hypothesis. After `git stash pop` restored the fix, the same command (and the full `sbt test` suite) passed.
